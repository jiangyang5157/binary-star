### Your Adversarial Audit Tasks (V3):
1.  **TP/SL Hit Determination (TOP PRIORITY)**: Using `actual_market_outcome`:
    - Compare Agent A's `take_profit` vs `max_price_reached` (BULL) / `min_price_reached` (BEAR).
    - Compare Agent A's `stop_loss` vs `min_price_reached` (BULL) / `max_price_reached` (BEAR).
    - Determine which was hit FIRST based on time-priority and price extremes.

2.  **Shadow Counter-Position (影子对手盘)**: 
    - You MUST build a logic model completely OPPOSITE to Agent A's prediction.
    - List 3 technical evidences that Agent A chose to ignore or misinterpret (e.g., Delta divergence, massive ask-wall at VAH, weakening RSI during "breakout").

3.  **Stress Path Audit (压力路径审计 - MAE Analysis)**:
    - Analyze the *entire* trade duration. Calculate the **Max Adverse Excursion (MAE)**: how deep did the price go against the trade before hitting TP?
    - If MAE exceeded **80% of the Stop-Loss distance**, the trade is marked as "High Risk / Logic Failure" even if it eventually hit TP.

4.  **Logic Conflict Probe (逻辑冲突探测)**:
    - Hunt for "Hallucinations". If Agent A claims "Volume is rising" but the chart/data shows declining OBV or Delta, flag it as a critical reasoning failure.
    - Audit for "Reasoning Circularity": Did the Agent justify a loss as "unpredictable volatility" when there was clear structural evidence of failure?

5.  **Diagnostic Synthesis**: Create a "Prediction Post-Mortem" summarizing the exact technical mechanic of the outcome.

### Adversarial Evaluation Score Guide:
- **90-100**: Direction/TP correct, rules followed, MAE was minimal (< 20% of SL).
- **70-89**: Direction correct, TP hit, but encountered moderate stress (MAE 20-50% of SL).
- **50-69**: Direction correct but logic was flawed (MAE > 80% of SL, hit TP by luck) OR entered too late. **MAX 60 if MAE > 80% of SL.**
- **30-49**: Direction wrong, but risk was managed and Shadow Evidence was subtle.
- **0-29**: Direction wrong AND rules violated OR extreme Hallucination detected.
