## CRITICAL Findings

### [D5] Anonymous Users Granted All Permissions Including Trading
- **File**: `config/auth/users.json:13-14`
- **Severity**: CRITICAL
- **Description**: The `anonymous` role is granted every permission: `view_active_sessions`, `run_new_session`, `run_sniper`, and `run_backtest`. Any unauthenticated request can start/stop the sniper daemon, run backtests, and trigger trading sessions.
- **Impact**: Anyone who can reach the dashboard port can execute real financial trades on Binance. The entire authorization framework is bypassed by design.
- **Recommendation**: Remove `run_sniper`, `run_backtest`, and `run_new_session` from the `anonymous` role. Require explicit authenticated user IDs for any operation that places trades or consumes system resources.

### [D5] Missing Authentication on 8 API Endpoints
- **Files**: `src/dashboard/api/audits.py:61,180,240,302,315,421`, `src/dashboard/api/sessions.py:76,163`
- **Severity**: CRITICAL
- **Description**: Eight API endpoints across `audits.py` and `sessions.py` have no `Depends(_require(...))` auth guard. `/api/audits/run` triggers a multi-threaded audit process consuming CPU and external API calls with zero authorization.
- **Impact**: Unauthenticated access to full trading performance history, trade records, system configuration, and the ability to trigger resource-intensive processes.
- **Recommendation**: Add `Depends(_require(...))` auth guards to every endpoint, matching the pattern already used in `session_run.py`, `sniper_run.py`, and `backtest.py`.

### [D5] Path Traversal in Audit File Retrieval
- **File**: `src/dashboard/api/audits.py:306`
- **Severity**: CRITICAL
- **Description**: `get_audit` constructs `path = Path(data_root) / "audits" / filename` with user-controlled `filename` from the URL path. No `.resolve()` or prefix validation — unlike `get_session` in `sessions.py:167-170` which does both.
- **Impact**: Arbitrary file read on the server via `GET /api/audits/../../.env` or similar. Can exfiltrate API keys, config, and user data.
- **Recommendation**: Apply the same pattern from `sessions.py:167-170`: use `.resolve()`, then verify `str(filepath).startswith(str(audits_dir) + os.sep)`.

### [D2] Cross-Agent Prompt Injection via Unsanitized Debate History
- **File**: `src/agent/debate_loop.py:121-126`, `config/prompts/session.md:10`, `config/prompts/critic.md:12`
- **Severity**: CRITICAL
- **Description**: Full LLM-generated output (`reasoning_chain`, `audit_evidence`, `critic_summary`) from one agent round is packed into `debate_history` and re-injected as `{debate_history_json}` into the peer agent's prompt for the next round — with no sanitization or instruction-boundary detection. A compromised/hallucinated LLM response containing embedded instructions would be read as legitimate context.
- **Impact**: An agent could inject meta-instructions that subvert the peer agent's role constraints, potentially causing unauthorized trade execution.
- **Recommendation**: (1) Strip `reasoning_chain` and `audit_evidence` from all rounds before passing to the peer. (2) Wrap history in a `<factual_record>` XML tag with instructions that nothing inside it is an instruction. (3) Validate `last_plan` JSON schema before injection.

### [D4] Emergency Market Close Failure Leaves Position Naked Permanently
- **File**: `src/agent/order_executor.py:324,338,372,441,509,528,533,615`
- **Severity**: CRITICAL
- **Description**: In 8 code paths, `execute_market_close()` is called but its boolean return value is ignored. After `cancel_all_symbol_orders()` removes existing OCO protection, a failed `execute_market_close()` still causes the caller to return `{}` (clearing `trade_state`). The SniperDaemon stops guardian checks for that symbol, but the actual position remains open on the exchange with zero protection.
- **Impact**: Naked position that the robot has forgotten. The guardian sees no `trade_state["direction"]` and returns immediately. If the market moves against the position, loss is unbounded. This is a capital-loss severity bug.
- **Recommendation**: Verify the position was actually closed via `get_symbol_position()` in a retry loop before clearing `trade_state`. Never return `{}` unless `abs(pos.net_qty) < tolerance` is confirmed.

### [D4] Direction Conflict in guardian_check Leaves Stale OCO Orders
- **File**: `src/agent/order_executor.py:303-314`
- **Severity**: CRITICAL
- **Description**: When `guardian_check()` detects an orientation conflict (e.g., `trade_state` says LONG but `net_qty` is negative), it logs a warning and returns `trade_state` unchanged — with zero cleanup. The stale OCO from the old direction remains live on the exchange.
- **Impact**: Stale SELL OCO on a SHORT position would increase the short on trigger (doubling intended size). The old SL becomes a bearish add-on to an already short position. Capital loss risk.
- **Recommendation**: Call `cancel_all_symbol_orders()` in the direction-conflict path before returning. Do not leave stale exit orders on the book.

## HIGH Findings

### [D1] Unconditional `set_triggered()` Resets Cooldown During Active Positions, Creating Dead Window After Exit
- **File**: `run_sniper.py:361`
- **Severity**: HIGH
- **Description**: `set_triggered(result)` is called unconditionally for every symbol whose trigger fires — including when `has_active` is True and the AI session is explicitly skipped. This resets `last_trigger_time` and `last_trigger_score` on every pulse during an active position. After the position closes, cooldown restarts from the last `set_triggered` call, blocking re-entry for the full cooldown duration.
- **Impact**: Systematic missed trading opportunities after position closure. The system can lock itself out of the market for extended periods.
- **Recommendation**: Move `set_triggered(result)` inside the `if self.session_engines.get(sym) and not has_active:` block.

### [D1] `_derive_price_dynamics` Lacks Empty-DataFrame Safety Guard
- **File**: `src/analyzer/market_observer.py:441,460,478-479`
- **Severity**: HIGH
- **Description**: `_derive_price_dynamics()` immediately accesses `m_df.iloc[-1]`, `m_df['atr'].iloc[-1]`, and `n_df['atr'].iloc[-1]` without checking if DataFrames are empty. Same vulnerability in `_derive_anchors()` with `df['close'].iloc[-1]` and `df['atr'].iloc[-1]`.
- **Impact**: Empty DataFrame raises `IndexError`, crashing the entire observation/scout cycle. This is in the most critical data pipeline path.
- **Recommendation**: Add guard clauses at the top of both functions to return empty/sentinel dicts if the DataFrame is empty.

### [D1] `SniperTrigger` Instantiated Without `symbol` in `SniperSampler` Causes `None` in Logging
- **File**: `src/analyzer/simulation_sampler.py:42`
- **Severity**: HIGH
- **Description**: `SniperTrigger` is initialized without a `symbol` argument. Throughout the trigger module, logging uses `self.symbol` producing `[None] SIGNAL DIAG`. Cross-symbol correlation lookup `CROSS_CORRELATIONS.get(self.symbol, 0.30)` silently defaults to an uncalibrated 0.30.
- **Impact**: All trigger diagnostics are logged under `[None]`, making sampling-mode debugging impossible. Cross-correlation values are wrong, skewing backtest accuracy.
- **Recommendation**: Pass `symbol` to the `SniperTrigger` constructor in `simulation_sampler.py:42`.

### [D2] Visual "Physical Ground-Truth" Claim Triggers Hallucinations
- **File**: `config/prompts/session.md:11`, `config/prompts/critic.md:13`
- **Severity**: HIGH
- **Description**: Both prompts state chart images "provide the physical ground-truth of market structure" — treating pixel-based chart renderings as more authoritative than numerical telemetry. LLMs are notoriously unreliable at extracting precise coordinates from chart images.
- **Impact**: Agent may hallucinate support/resistance levels from chart images that contradict actual numeric data, trusting the visual "ground-truth" over numbers.
- **Recommendation**: Replace "physical ground-truth" with "visual reference for qualitative pattern context." Add explicit priority: "Numerical telemetry in Observation Content is the authoritative source."

### [D2] Vision Instructions Active for Providers and Code Paths That Do Not Supply Images
- **File**: `src/agent/binary_star_orchestrator.py:447-448`, `src/infrastructure/ai/_openai_helpers.py:47-50`
- **Severity**: HIGH
- **Description**: `shared_instruction` always includes the `VISUAL_CONTEXT_INTERPRETATION` section even when `visual_parts` is empty or the active provider doesn't support vision. OpenAI-compatible adapters silently skip `VisualPart` items, so the LLM is told to expect visual evidence but receives none.
- **Impact**: The LLM fabricates visual observations to satisfy the prompt's expectation, introducing hallucinated market structure analysis.
- **Recommendation**: Gate the `VISUAL_CONTEXT_INTERPRETATION` section behind a conditional. Add `"[NO VISUAL DATA PROVIDED]"` preamble when images are absent.

### [D2] Conflicting NEUTRAL Directives Create Risk-Taking Bias
- **File**: `config/prompts/session.md:67,102-104`
- **Severity**: HIGH
- **Description**: "Cowardice Veto" (line 67) pushes toward directional plans, while "Zero-Score Override" (lines 102-104) mandates NEUTRAL → score 0. The combination creates a perverse incentive: NEUTRAL incurs a zero confidence score (which looks bad), while directional plans avoid this penalty.
- **Impact**: Session Agent is nudged toward submitting unbacked directional plans to avoid zero-score embarrassment. The Critic never audits whether a directional plan should have been NEUTRAL.
- **Recommendation**: Remove the zero-score penalty for NEUTRAL. Give NEUTRAL a default score floor (e.g., 50) — higher than a bad directional plan but lower than a validated one.

### [D2] Missing Few-Shot Examples for Complex Confidence Calculus
- **File**: `config/prompts/session.md:101-111`
- **Severity**: HIGH
- **Description**: The confidence score is computed via a penalty-based system across 3 dimensions with 14 distinct deduction rules — the single most complex arithmetic operation asked of the LLM — with zero worked examples. Instructions use abstract logic like "Subtract -10 if the entry sits in a volume vacuum without a proximal HVN."
- **Impact**: Different LLM calls on identical data produce divergent scores. Scoring becomes non-deterministic, undermining its purpose as a "Structural Hardness Score."
- **Recommendation**: Add 2-3 worked examples: (a) pristine BULLISH plan with full score, (b) marginal plan with triggered deductions, (c) NEUTRAL plan.

### [D2] Unstructured `reasoning_chain` Field Risks Non-Deterministic Audit Trail
- **File**: `config/prompts/session.md:129`
- **Severity**: HIGH
- **Description**: `reasoning_chain` has no structure specification — no length limit, no sub-fields, no machine-parseable format. The Confidence Calculus math is expected to be "clearly shown" but has no machine-readable output slot.
- **Impact**: Downstream systems (AuditController, Evolver) cannot reliably parse deduction decisions. The system cannot distinguish "score was computed correctly" from "score was hallucinated."
- **Recommendation**: Add a `score_breakdown` sub-object with per-dimension raw scores and deduction lists. Keep `reasoning_chain` for narrative but make scoring machine-parseable.

### [D2] Opinion String Case Sensitivity Can Cause Silent Logic Failures
- **File**: `config/prompts/critic.md:27-29`, `config/prompts/session.md:119`
- **Severity**: HIGH
- **Description**: Critic macros use strict string equality (`last_plan.opinion == "BULLISH"`). No instruction to the Session agent requiring exact case/whitespace, and no Python-level normalization in `debate_loop.py`. If Session outputs `"Bullish"` or `"BULLISH "`, ALL Critic macros silently evaluate to false.
- **Impact**: Single-point-of-failure for the entire adversarial safety net. A case deviation causes silent PASS for plans that should have been flagged.
- **Recommendation**: (1) Add explicit instruction for exact uppercase strings. (2) Add Python-level normalization: `last_plan["opinion"] = last_plan.get("opinion", "NEUTRAL").strip().upper()`.

### [D3] HTTP Connection Leak in AI Client Adapters (All Providers)
- **File**: `src/infrastructure/ai/_openai_helpers.py:174-196`, `src/infrastructure/ai/gemini_adapter.py:22-25`, `src/infrastructure/ai_client.py:45-79`
- **Severity**: HIGH
- **Description**: None of the three AI client implementations provide a `close()` method. `AbstractAIClient` defines no lifecycle hooks. Lazy-created `openai.OpenAI` instances manage internal `httpx.Client` connection pools that are never released.
- **Impact**: Resource exhaustion in long-running processes (dashboard, backtest runner). Leaked connections accumulate file descriptors, eventually hitting OS limits and crashing with "Too many open files."
- **Recommendation**: Add `close()` to `AbstractAIClient` and implement in each adapter. Call `self.client.close()` when agents are done.

### [D3] Module-Level Mutable State With Concurrent Read/Write in Dashboard
- **File**: `src/dashboard/server.py:51,84,133-134,87-96`
- **Severity**: HIGH
- **Description**: `_users_permissions` dict is a module-level mutable variable reassigned from request handlers (`/live`, `/development` endpoints) while being read by `_get_user_permissions()` on every API request — all without locks.
- **Impact**: Race condition could cause KeyError or incorrect permission denials, blocking legitimate users or granting unintended access.
- **Recommendation**: Use `threading.Lock`, `copy.deepcopy` on read, or move to `app.state` with proper locking.

### [D3] Bracket Config Access on Potentially Empty Dict in BinanceFuturesClient
- **File**: `src/infrastructure/binance/client.py:60-62`
- **Severity**: HIGH
- **Description**: Defensive `.get()` with empty-dict fallbacks protects `network_cfg` access, but immediately after, bracket notation accesses `binance_net['api_timeout_seconds']` and `binance_net['retry_count']`. Missing keys raise `KeyError`, crashing the entire client constructor.
- **Impact**: A single missing optional config key crashes the exchange client at construction time. If this happens during dashboard startup, the entire uvicorn process fails.
- **Recommendation**: Validate all keys exist using `.get()` with defaults, or add a validation step raising descriptive `ConfigurationError`.

### [D4] Cancel-Then-Replace Creates a Naked Window in Trailing Stop Migration
- **File**: `src/agent/order_executor.py:499-529`
- **Severity**: HIGH
- **Description**: `_migrate_trailing_stop()` cancels all orders first, then places the new OCO. Between these two synchronous HTTP calls, the position is completely naked. If the replace call fails, the emergency path has the same return-value-ignored problem (CRITICAL finding #5).
- **Impact**: In volatile markets, a 200-800ms window with no protection can severely damage the position. If the replace fails, the position is abandoned.
- **Recommendation**: Place the new OCO before cancelling the old one. Only cancel after the new one is confirmed. This eliminates the naked window entirely.

### [D4] Stale OCO Remains After Partial Fill Reduces Position
- **File**: `src/agent/order_executor.py:270-274,346-372`
- **Severity**: HIGH
- **Description**: After partial SL fill, `has_oco` evaluates to True (the remaining SL leg is still on the book), entering Case 4 (trailing stop migration) instead of Case 3 (fresh OCO). The stale OCO quantity still reflects the original full position size, not the reduced position.
- **Impact**: Over-closing by up to the difference between original and remaining position size on the next migration.
- **Recommendation**: In Case 4, re-verify OCO quantity matches current position size. If mismatched, cancel and re-place even if no trailing migration is needed.

### [D4] Pre-Trigger Gate Silently Skipped When No HVN Anchors Exist
- **File**: `src/sniper/trigger.py:396-404`
- **Severity**: HIGH
- **Description**: When `anchors_below`/`anchors_above` is empty or has no HVN entries, `nearest_hvn` is `None`, and the entire structural distance check is skipped. The gate returns `"PASS"` without any volume-profile validation.
- **Impact**: In unstable markets, the trigger fires without structural validation. The AI consumes tokens and may produce an entry plan with no viable anchor.
- **Recommendation**: When `nearest_hvn` is `None`, compute fallback from `vah`/`val`/`poc`. Only skip the check if ALL volume-profile fields are absent — then return `"WEAK_PASS"`.

### [D4] get_ticker_price Returns 0.0 on Error Creating Silent Zero-Price Cascade
- **File**: `src/infrastructure/binance/margin_client.py:101-111`, `src/agent/order_executor.py:698-700`
- **Severity**: HIGH
- **Description**: `get_ticker_price()` returns `0.0` on API error, not `None`. In `_calculate_target_qty`, if the benchmark ticker fails: `total_equity_usdt = total_equity_btc * 0.0 = 0.0`, causing `target_qty = min_order_qty`. The trade proceeds at minimum size regardless of actual equity.
- **Impact**: Position sizing bypassed entirely if benchmark ticker is temporarily down. Trade executes at minimum size with no risk calculation.
- **Recommendation**: Return `None` instead of `0.0` on error. Use cached ticker price as fallback when live fetch fails.

## MEDIUM Findings

### [D1] Off-by-One in Normalized Velocity Calculation
- **File**: `src/analyzer/market_observer.py:460`
- **Severity**: MEDIUM
- **Description**: `net_displacement = abs(m_df['close'].iloc[-1] - m_df['close'].iloc[-trend_lookback])` measures change from candle -N to -1 — N candles but only N-1 intervals. Dividing by `trend_lookback` (N) underestimates per-bar velocity by ~1/N (about 7% for a 14-bar lookback).
- **Impact**: Inflated projected holding/waiting times. Guardian holds positions longer than warranted; entry-order timeout more lenient.
- **Recommendation**: Use `iloc[-(trend_lookback + 1)]` or divide by `(trend_lookback - 1)`.

### [D1] `refine()` Calls Analysis Functions With No Guard for NaN Indicator Columns
- **File**: `src/analyzer/market_observer.py:429-431,443-467`
- **Severity**: MEDIUM
- **Description**: `_derive_price_dynamics()` retrieves `atr_m = m_df['atr'].iloc[-1]` which can be NaN when DataFrames have insufficient candle depth for rolling windows. NaN flows into `volatility_intensity_index` and downstream trigger comparisons — all silently evaluating to False.
- **Impact**: Market regime detection degrades silently. VIX thresholds never trigger; volatility signals are suppressed without error messages.
- **Recommendation**: Add NaN guards in `_derive_price_dynamics()`: replace NaN values with sentinel defaults before returning.

### [D1] TopographyEngine.reconstruct() Closes Shared Exchange Client
- **File**: `src/analyzer/topography_engine.py:80`
- **Severity**: MEDIUM
- **Description**: `reconstruct()` calls `self.exchange_client.close()` in `finally` block unconditionally. If the client was injected as a shared instance, this terminates the shared connection for all subsequent users.
- **Impact**: Any code calling `reconstruct()` more than once on the same instance crashes on the second call. Violates principle of least surprise.
- **Recommendation**: Do not close the client in `reconstruct()`. Caller owns the client lifecycle.

### [D1] `_build_entry_suggestion` Falls Through to Bullish Bias for NEUTRAL Direction
- **File**: `src/sniper/trigger.py:618-623`
- **Severity**: MEDIUM
- **Description**: When `direction` is NEUTRAL (possible from `_detect_squeeze` with `Direction.NEUTRAL`), the entry suggestion falls through to the else branch implying bullish bias ("dip" / "below"), contradicting the situation brief's stated NEUTRAL direction.
- **Impact**: SessionAgent receives a NEUTRAL briefing with a bullish-biased entry zone. May confuse the LLM or encourage unauthorized trade.
- **Recommendation**: Add a NEUTRAL branch returning non-directional suggestion (`"type": "neutral_await"`, `"target_area": "await directional confirmation"`).

### [D1] `_detect_cvd_momentum` Fires on First Pulse Without Persistence Check
- **File**: `src/sniper/trigger.py:666-669`
- **Severity**: MEDIUM
- **Description**: On the first pulse after startup, any CVD exceeding threshold triggers a signal at full strength without persistence check. After the second pulse, the growth ratio filter engages requiring multiplicative growth.
- **Impact**: First pulse over-generates signals, potentially triggering immediate AI session on daemon start, wasting tokens on transient noise.
- **Recommendation**: Skip CVD momentum on the first pulse when `prev_metrics` is None.

### [D2] Evolver Prompt Mutations Lack Safety Guardrails
- **File**: `config/prompts/evolver.md:92-103`, `src/utils/evolution_utils.py:96-177`
- **Severity**: MEDIUM
- **Description**: Evolver-generated `semantic_refinement` patches are applied directly to prompt files without validation. `anchor_text` matching is done with a whitespace-flexible regex that could match unintended substrings. No rollback capability beyond batch sandbox.
- **Impact**: A single evolutionary step could corrupt a prompt file or inject malformed instructions. Structural damage to prompts undetected until runtime.
- **Recommendation**: Validate `anchor_text` matches exactly once. Add schema validation for `replaced_with`. Always backup prompt files before applying mutations.

### [D2] JSON Mode Instruction Fragilely Appended to System Prompt
- **File**: `src/infrastructure/ai/_openai_helpers.py:13-27`, `src/agent/base_agent.py:141`
- **Severity**: MEDIUM
- **Description**: When `use_json_mode = True`, a `JSON_HINT` ("You MUST respond ONLY with a valid JSON object...") is appended to the system instruction. This blanket directive can override the prompt's instruction to produce reasoning before JSON.
- **Impact**: Critic may suppress reasoning output, producing JSON with empty or hallucinated `audit_evidence` and `critic_summary` fields.
- **Recommendation**: Remove `JSON_HINT` append. Per-prompt output schemas already specify JSON. Use native JSON mode only for final synthesis, not the Critic audit.

### [D3] Config-Dependent Imports Executed at Class Instantiation Time
- **File**: `src/analyzer/simulation_sampler.py:38-42`
- **Severity**: MEDIUM
- **Description**: `SniperSampler.__init__()` performs lazy imports of `SniperScout` and `SniperTrigger`, which trigger cascading config loading at import time. Broken config files produce `ImportError`/`FileNotFoundError` wrapped in the caller's try/except.
- **Impact**: A YAML syntax error prevents the entire backtest sampling system from functioning. API returns 500 with confusing error message.
- **Recommendation**: Move config loading out of `__init__` into explicit `initialize()` method. Or use dependency injection.

### [D3] Naive/Aware Datetime Mixing Risk in `_is_stale` and `_parse_session_timestamp`
- **File**: `src/dashboard/api/backtest.py:106-107`, `src/dashboard/api/sessions.py:69`
- **Severity**: MEDIUM
- **Description**: `_is_stale()` subtracts a potentially naive `datetime` from an aware `datetime.now(timezone.utc)`. Python 3.13 raises `TypeError`. The broad `except Exception: return True` masks this silently.
- **Impact**: Corrupt status file silently causes all backtest results to appear stale. Operator sees no data and doesn't know why.
- **Recommendation**: Always use `parse_iso_string_to_utc` from `datetime_utils`. Add defensive `if started.tzinfo is None: started = started.replace(tzinfo=timezone.utc)`.

### [D4] OCO and LIMIT Order `sideEffectType` Always `MARGIN_BUY` Causing Potential Unwanted Borrowing
- **File**: `src/infrastructure/binance/margin_client.py:202,221`
- **Severity**: MEDIUM
- **Description**: All order placements hardcode `sideEffectType="MARGIN_BUY"`, which auto-borrows if insufficient free balance. For market-close BUY orders on SHORT positions, this could borrow additional USDT rather than using proceeds from the initial short sale.
- **Impact**: Position closed but new debt created. Margin level may drop unexpectedly.
- **Recommendation**: Use `NO_SIDE_EFFECT` for all exit orders. Reserve `MARGIN_BUY` for new entry orders only.

### [D4] `noise_factor` in Confluence Engine Suppresses Valid Directional Signals at High Concurrence
- **File**: `src/sniper/trigger.py:156`
- **Severity**: MEDIUM
- **Description**: `noise_factor = 1.0 - (bullish_score * bearish_score)` penalizes product of scores. When both sides are strong (absorption patterns), the dominant side is suppressed to near-zero despite having genuine dominance.
- **Impact**: Legitimate absorption setups suppressed. Valid trades missed.
- **Recommendation**: Replace with ratio-based: `noise_factor = 1.0 - min(bullish_score, bearish_score) / max(bullish_score, bearish_score + epsilon)`.

### [D4] Adaptive Cooldown Decay Half-Lives From Config Never Applied
- **File**: `src/sniper/trigger.py:66-71,684,720,756,786,829,860,910,948,1011,1070,1122,1157,1190`
- **Severity**: MEDIUM
- **Description**: `global_config.yaml` defines decay half-lives in `signal_stack.decay`, but every `SignalCard` constructor hardcodes its `decay_half_life_minutes`. Config values are never read.
- **Impact**: Operators who modify config expect half-lives to change — they don't. Maintenance trap and tuning blind spot.
- **Recommendation**: Load decay config in `SniperTrigger.__init__` and pass to each detector. Log active half-life values at startup.

### [D4] `_optimize_same_direction` Cancel Failure Silently Discards AI-Recommended TP/SL
- **File**: `src/agent/order_executor.py:585-587`
- **Severity**: MEDIUM
- **Description**: When cancel fails, method returns `True` (position protected) but the new TP/SL from AI opinion is silently discarded. No log distinguishes "optimization applied" from "optimization skipped due to cancel failure."
- **Impact**: Over consecutive cancel failures, AI's TP/SL continuously drifts from active OCO. Dashboard shows "optimization complete" but OCO is stale.
- **Recommendation**: Log WARNING with existing and proposed TP/SL when cancel fails. Persist `desired_tp`/`desired_sl` in `trade_state` for retry on subsequent pulses.

### [D5] Weak data_root Path Traversal Protection
- **File**: `src/dashboard/api/sessions.py:14-17`, `src/dashboard/api/audits.py:14-18`
- **Severity**: MEDIUM
- **Description**: `_resolve_data_root()` only checks if `".."` is a substring of the query parameter. URL-encoded `..` (`%2e%2e`), double-encoding, or absolute paths bypass the check. `data_root` is then passed to `subprocess.Popen()` via `-p` argument.
- **Impact**: Path traversal to read/write outside intended data directory. Potential subprocess argument injection vector.
- **Recommendation**: Use `Path(data_root).resolve()` and verify it starts with allowed root. Use whitelist, not blacklist.

### [D5] No Rate Limiting on Any API Endpoint
- **Files**: All dashboard API routers
- **Severity**: MEDIUM
- **Description**: None of the API endpoints implement rate limiting. `POST` endpoints that spawn subprocesses or call external APIs (Binance, LLM providers) have no protection against rapid-fire requests.
- **Impact**: An attacker or buggy frontend could spawn hundreds of subprocesses, exhausting system resources, exceeding LLM API rate limits, potentially causing Binance API bans.
- **Recommendation**: Add `slowapi` or custom middleware to rate-limit by IP. Set aggressive limits on POST endpoints that spawn subprocesses.

### [D5] Missing Jinja2 Auto-Escaping in Dashboard Templates
- **File**: `src/dashboard/server.py:46`

## LOW Findings

### [D1] `liquidation_estimator._cluster_points` Normalizes Intensity Per-Observation
- **File**: `src/analyzer/liquidation_estimator.py:190-193`
- **Severity**: LOW
- **Description**: Intensity normalized by `max_intensity + 1e-9` within each observation. Weak clusters in one session get `intensity=1.0`, same as strong clusters in another — cross-session comparisons impossible.
- **Recommendation**: Keep raw intensity alongside normalized value, or document per-observation normalization.

### [D1] `price_bins` Uses Left-Edge Price for Profile Data
- **File**: `src/analyzer/volume_profile.py:133,147-148`
- **Severity**: LOW
- **Description**: VAH and profile data use `price_bins[i]` (left edge) instead of bin center. Systematic one-bin-width bias in volume profile calculations.
- **Recommendation**: Use bin center: `price_bins[i] + (price_bins[1] - price_bins[0]) / 2`.

### [D1] `run_sniper.py` Closure Captures `sym` by Reference
- **File**: `run_sniper.py:287-337`
- **Severity**: LOW
- **Description**: `_sniper_progress` closure captures `sym` from enclosing loop scope. Currently benign (synchronous execution) but would break if `execute_cycle` became async.
- **Recommendation**: Bind `sym` at closure creation: `def _sniper_progress(..., _sym=sym):`.

### [D2] `safe_format` Missing-Key Regex Does Not Catch Variable-Pattern Variations
- **File**: `src/agent/base_agent.py:108`
- **Severity**: LOW
- **Description**: Regex `r'\{(\w+)\}'` doesn't match `{dotted.path}` or `{snake_case_123}`. Current templates use only simple names, but future template authors could bypass the safety check.
- **Recommendation**: Replace regex with `string.Formatter().parse(template)`.

### [D3] Singleton Module-Level Mutable Object in MathTools
- **File**: `src/utils/math_utils.py:450-453`
- **Severity**: LOW
- **Description**: Module-level singleton with monkey-patched attributes. Any future mutation to `MathTools.xxx` affects all callers globally.
- **Recommendation**: Replace with plain module-level functions. Deprecate the `MathTools.xxx()` access pattern.

### [D3] Bare `except Exception: pass` on Session Close in BinanceClient
- **File**: `src/infrastructure/binance/client.py:354`
- **Severity**: LOW
- **Description**: `close()` wraps session close in bare `except: pass`. If session is None, the `AttributeError` is swallowed and the caller believes cleanup succeeded.
- **Recommendation**: Log at DEBUG level: `logger.debug("session close failed | error=%s", e)`.

### [D4] Cross-Symbol Correlation Defaults to 0.30 for Undefined Symbols
- **File**: `src/sniper/trigger.py:214-217`, `run_sniper.py:226-228`
- **Severity**: LOW
- **Description**: `CROSS_CORRELATIONS.get(follower_sym, 0.30)` defaults to 0.30 for any undefined symbol — higher than XAUT's calibrated 0.40. False-positive triggers on uncalibrated symbols.
- **Recommendation**: Default to 0.0 for undefined symbols. Log warning when undefined symbol receives correlation query.

### [D4] `_guardian_check` Passes Stale ATR From Previous Pulse
- **File**: `run_sniper.py:477-480`
- **Severity**: LOW
- **Description**: ATR passed to guardian comes from `prev_metrics` — up to 2 minutes old. In extreme volatility, trailing stop migrates at wrong level by 5-10% of ATR.
- **Recommendation**: Compute current ATR inline from recent klines or document the one-pulse-lag behavior.

### [D5] User ID Transmitted as URL Query Parameter
- **File**: `src/dashboard/server.py:101`
- **Severity**: LOW
- **Description**: `user` parameter passed in URL query string, exposing it to browser history, server logs, and `Referer` headers.
- **Recommendation**: Move authentication to cookie or `Authorization` header.

### [D6] Dashboard Modules (7 files) Have Zero Test Coverage
- **File**: `src/dashboard/` (entire directory)
- **Severity**: LOW
- **Description**: Entire dashboard stack untested. `api/audits.py` uses `ThreadPoolExecutor` — the most thread-unsafe code in the project — with no tests.
- **Recommendation**: Add basic smoke tests for API endpoints using FastAPI `TestClient`.
