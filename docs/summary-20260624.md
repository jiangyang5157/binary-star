### H1. VolumeProfile — `df.iterrows()` on Every Hot-Path Observation
**File:** `src/analyzer/volume_profile.py:84-96`  
**Severity:** HIGH — CPU waste  

Python `iterrows()` loop over 500+ candles × 300 bins on every observation. Across backtests with thousands of sessions, this dominates CPU. Vectorizable with `np.digitize` + `np.add.at`.

---

### H2. Same Kline Processing Executed Twice Per Observation
**File:** `src/analyzer/market_observer.py:411-412` and `697-698`  
**Severity:** HIGH — 2× CPU waste  

`process_klines()` is called in `refine()` AND again in `_generate_snapshots()` for the same raw data. This doubles ATR recalculation cost and exacerbates the `iterrows()` problem (H1).

---

### H10. Time-Stop Uses Stale Holding-Time Projection
**File:** `src/agent/order_executor.py:375-387`  
**Severity:** HIGH — Premature/overdue position close  

`projected_holding_hours` is computed ONCE when the trade opens. If volatility changes, the physical time to TP changes but the time-stop deadline doesn't. A volatility spike means the trade should close faster; a volatility collapse means it should run longer. Neither adjustment happens.

### H13. Fast-Pass Bypass Can Skip Critical Audits After Market Regime Change
**File:** `src/agent/debate_loop.py:165-226`  
**Severity:** HIGH — Missed trade validation  

The `_evaluate_critic_fast_pass` amnesty clause returns PASS when a prior TERMINAL veto exists, skipping the critic entirely. If the market has materially changed since the TERMINAL veto round, the NEUTRAL decision gets no adversarial review.

## Configuration & YAML Review

### `strategy_config.yaml`

- **CFG1.** `volume_profile_value_area_width: 0.86` — 86% Value Area is unusually wide (standard is 68-70%). This captures more volume but makes VAH/VAL boundaries less significant as support/resistance.
- **CFG2.** `squeeze_threshold: 1.0` — BB_width / KC_width < 1.0 means Bollinger Bands are inside Keltner Channels. This is the standard definition. The `squeeze_audit_threshold: 0.85` is a separate, tighter threshold used only by the Critic's fast-pass check.
- **CFG3.** `min_rr_ranging: 1` and `min_rr_trending: 1.2` — Ranging allows 1:1 RR (50% win rate = breakeven). Trending requires 1.2:1. With `chaos_rr_discount: 0.35`, chaos minimum becomes 0.65 for ranging and 0.78 for trending — appropriately aggressive for survival mode.
- **CFG4.** `max_holding_hours: 72.0` with `temporal_weight_climax: 0.3` → effective max hold in climax = 21.6 hours. With `time_stop_multiplier: 1.5` → time-stop at 32.4 hours. Reasonable.
- **CFG5.** `wick_skew_fallback: 0.5` — When a candle has zero range (doji), the fallback assumes neutral wick skew. A doji at a boundary is actually a significant signal — a more informative fallback would look at the prior candle or use `None`.

### `global_config.yaml`

- **CFG6.** `active_provider: "deepseek"` — DeepSeek is used as the primary LLM. DeepSeek does NOT support vision (confirmed by `supports_vision: false` in the config). This means chart images (VisualPart) are **never sent to the LLM**. The prompt instructions reference `[VISUAL_CONTEXT]` extensively, but the agent never actually sees the charts. **This is a significant strategy gap** — the Session and Critic agents are making decisions without the visual ground truth they're instructed to use.
- **CFG7.** `max_rounds: 2` — Only 2 debate rounds. This means: Round 1 (planning + audit), Round 2 (revision + audit). If Round 2 fails, cold synthesis produces the final decision. This is tight but cost-efficient. The risk is that complex market setups need 3+ rounds to converge.
- **CFG8.** `risk_per_trade: 0.004` (0.4% of equity). Conservative. With 0.4% risk per trade and a 1.2 RR minimum, the breakeven win rate is ~45%.
- **CFG9.** `net_qty_tolerance: 0.00001` — This is the minimum BTC quantity to consider a position "FLAT." At $100k/BTC, this is $1. Reasonable, but if BTC price falls to $10k, this becomes $0.10 which might be too tight (rounding errors could show phantom positions).
- **CFG10.** Sniper `pulse_interval_minutes: 2.0` with `pulse_cooldown_multiplier: 2.5` → cooldown = 5 minutes. With `chaos_cooldown_multiplier: 2.0` → chaos cooldown = 10 minutes. The `state_lockout_hours: 8.0` blocks retriggers for a full trading session.
- **CFG11.** Guardian trailing levels: Level 1 at 1.5 ATR profit (SL→breakeven), Level 2 at 2.5 ATR (SL→entry+0.5 ATR), Level 3 at 4.0 ATR (SL→entry+1.5 ATR). These are aggressive — a position needs to move 1.5× ATR before even reaching breakeven. Many trades will be stopped out before trailing activates.

### `symbol_config.yaml`

- **CFG12.** BTCUSDT `sl_slippage_buffer: 10.0` — A $10 buffer on BTC stop-loss. At $100k BTC, this is 0.01% — very tight. During volatility, this could cause the stop limit to not fill.
- **CFG13.** XAUTUSDT `sl_slippage_buffer: 0.5` — $0.50 buffer on gold. At ~$3000 XAUT, this is 0.017% — similarly tight.
- **CFG14.** XAUTUSDT overrides: `cvd_divergence_tick_delta: 0.18` and `cvd_impulse_tick_delta: 0.25` (lower than BTC defaults of 0.22/0.35). This makes XAUT more sensitive to flow signals — appropriate for a thinner market.

## Architecture & Strategy Observations

### Strengths

1. **Dependency Injection Everywhere** — `AbstractAIClient`, `AbstractExchangeClient`, config dataclasses all injected. Clean testing seams.
2. **Frozen Dataclasses for Config** — Immutable configuration prevents accidental mutation.
3. **Domain Exception Hierarchy** — `SingularityError → AgentInferenceError → ...` enables precise error handling.
4. **Adversarial Debate Architecture** — Session→Critic with MathFactChecker verification. Genuinely innovative.
5. **Guardian System** — Progressive trailing stop + time-based stop + emergency fallback. Professionally designed.
6. **Provider-Agnostic AI Layer** — `VisualPart`, `AIResponse`, `ToolCall` abstractions decouple agents from SDKs.
7. **Config Resolution Stack** — `base + symbol.overrides → resolved` is clean and testable.

### Architecture Risks

1. **Visual Context Disabled:** DeepSeek (`supports_vision: false`) means chart images are NEVER sent to the LLM. The Session and Critic prompts extensively reference `[VISUAL_CONTEXT]` but the agents operate text-only. The system is effectively running blind to the charts it generates.
2. **Single Point of Failure: AI Inference.** No fallback to deterministic strategy on provider outage.
3. **No Paper Trading / Fast Backtest Mode.** All backtesting goes through the full LLM pipeline, incurring API costs.
4. **Configuration Complexity:** ~150+ parameters across 4 YAML files. A single misconfigured value could cause financial loss.
5. **Binary Star Protocol Assumes Rational Debate.** If both agents hallucinate in the same direction, the adversarial safeguard fails. MathFactChecker only verifies arithmetic, not directional correctness.
6. **No Account-Level Drawdown Circuit Breaker.** The strategy intent says "LARGE DRAWDOWNS ARE ABSOLUTELY PROHIBITED" but the system has no mechanism to detect or enforce this.
