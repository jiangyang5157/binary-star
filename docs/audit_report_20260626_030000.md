## CRITICAL Findings (6)

### [D4-F1] Guardian Initial OCO Failure Leaves Position Naked
- **File**: `src/agent/order_executor.py:367-369`
- **Severity**: CRITICAL
- **Description**: In `guardian_check` Case 3 (position with no OCO), `place_oco_order` fails → code logs error but does NOT emergency-close. Every other failure path in the file correctly triggers `execute_market_close` — this one is the sole omission.
- **Impact**: Position completely unprotected until next pulse (up to 2 minutes). Unlimited-loss window in fast-moving markets.
- **Recommendation**: Add `self.client.execute_market_close(symbol)` and `return {}` immediately after line 368.

### [D1-F1] TriggerResult Unpacking Crash in SimulationSampler
- **File**: `src/analyzer/simulation_sampler.py:67`
- **Severity**: CRITICAL
- **Description**: `SniperTrigger.evaluate()` returns a `TriggerResult` dataclass, but line 67 tries to unpack it as a 3-tuple `(bool, str, str)`. This raises `TypeError: cannot unpack non-iterable TriggerResult object`.
- **Impact**: `SniperSampler.sample()` crashes on every call. The entire historical sampling pipeline is non-functional.
- **Recommendation**: Replace unpacking with attribute access: `result.triggered`, `result.gate_result`, `result.gate_reason`.

### [D1-F2] NaN Propagates Through `if atr` Truthiness Guard
- **File**: `src/analyzer/market_observer.py:478-484`
- **Severity**: CRITICAL
- **Description**: `bool(float("nan"))` evaluates to `True`. When ATR is NaN (insufficient kline data for rolling window), `if atr else 0` does NOT catch it. Division produces NaN that propagates into `structural_anchors`, then into SniperTrigger signal detection, producing NaN-weighted signals.
- **Impact**: NaN silently corrupts confluence engine calculations. NaN-weighted signals produce NaN product scores and nondeterministic trigger behavior.
- **Recommendation**: Use `if atr and not pd.isna(atr)` as the guard.

### [D5-F1] Missing Authentication on All Dashboard Write Endpoints
- **File**: `src/dashboard/api/sniper_run.py:64-66`, `session_run.py:141-142`, `backtest.py:395-396`
- **Severity**: CRITICAL
- **Description**: 6 write-capable REST endpoints (start/stop sniper, run sessions, run backtests) have ZERO authentication enforcement. Permission system exists but is only used for UI toggles in Jinja2 templates. Dashboard binds to `0.0.0.0:8080` by default. Anonymous role in `users.json` grants ALL permissions.
- **Impact**: Any network-accessible attacker can start/stop live trading, trigger real-money sessions, or launch compute-intensive backtests.
- **Recommendation**: Add FastAPI dependency-injected auth guards. Bind to `127.0.0.1` by default. Reduce anonymous role to read-only.

### [D4-F2] Missing TP/SL in Trade State — Silent Return, No Emergency Close
- **File**: `src/agent/order_executor.py:321-323`
- **Severity**: CRITICAL
- **Description**: When `guardian_check` detects a live position with no OCO AND no TP/SL in `trade_state`, it logs "Cannot protect!" and returns without closing the position. The position is left exposed indefinitely.
- **Impact**: A corrupted trade_state (crash recovery, partial deserialization) leaves a real position with no protective orders, potentially forever.
- **Recommendation**: Call `self.client.execute_market_close(symbol)` and `return {}`.

### [D3-F1] Bare `except: pass` Swallows All Errors in Shutdown Handlers
- **File**: `run_session.py:158, 181`
- **Severity**: CRITICAL
- **Description**: Two bare `except: pass` blocks in session engine's `finally` block and signal handler silently discard all exceptions from `delete_market_cache()`. The signal handler case (lines 177-182) could prevent `sys.exit(0)` from running.
- **Impact**: Stale Gemini cache entries accumulate silently, inflating API costs. Process termination can hang.
- **Recommendation**: Replace with `except Exception as e: logger.warning(...)`.

## HIGH Findings (17)

### Logic Errors

### [D1-F3] ATR NaN Reaches `scipy.signal.find_peaks`
- **File**: `src/analyzer/volume_profile.py:170, 185, 256`
- **Description**: ATR NaN passes the `atr <= 0` guard (NaN comparisons are always False), reaches `int(atr * min_node_gap_atr / bin_width)` causing undefined behavior in `find_peaks`.
- **Impact**: Either crash or spurious HVN/LVN detection for first `atr_period` candles.
- **Recommendation**: Add `if atr <= 0 or math.isnan(atr): return {"hvn": [], "lvn": []}`.

### [D1-F4] `find_nodes` Bin Width Uses Wrong Denominator
- **File**: `src/analyzer/volume_profile.py:184`
- **Description**: `bin_width = price_range / len(prices)` — `len(prices)` is count of non-zero-volume bins, not total bins (300). Price gaps cause over-narrow perceived bin width, causing `find_peaks` to detect false structural nodes.
- **Impact**: Over-detection of HVNs/LVNs → trigger fires on non-existent structure → poor trade geometry.
- **Recommendation**: Use `self.config.resolution_bins` (300) as denominator.

### Prompt Issues

### [D2-F1] Counter-Trend Prohibition Conflicts with Sweep & Fade Authorization
- **File**: `config/prompts/session.md:39, 61-64`
- **Description**: Line 39 says "DO NOT execute counter-trend trades against IS_TREND_STRONG." Lines 61-64 authorize "Sweep & Fade (Counter-Trend Reversal)." No exception clause on the prohibition.
- **Impact**: Agent cannot deterministically resolve whether to execute valid counter-trend setups during strong trends.
- **Recommendation**: Add explicit override: "unless the Sweep & Fade conditions at lines 61-64 are fully met."

### [D2-F2] No Positive-Value Validation on Price Outputs
- **File**: `config/prompts/session.md:118-128`, `math_fact_checker.py:46-48`, `order_executor.py:61-66`
- **Description**: OUTPUT_SCHEMA defines prices as "decimal" with no positivity/finite-number constraints. `float(tactical.get('entry', 0) or 0)` silently converts None to 0. No code-level validation between LLM output and exchange API.
- **Impact**: NaN, negative, zero, or infinite prices can reach the exchange client without correction.
- **Recommendation**: Add "MUST be a positive finite number" to schema. Add `math.isfinite()` guards in math_fact_checker and pre-submission validation in order_executor.

### System Bugs

### [D3-F2] BinanceFuturesClient.close() Is a No-Op — Connection Leaks
- **File**: `src/infrastructure/binance/client.py:344-346`
- **Description**: `close()` is literally `pass`. Underlying `UMFutures` SDK client maintains HTTP connection pools that are never released.
- **Impact**: Socket exhaustion in long-running daemons or batch backtesting.
- **Recommendation**: Implement proper `close()` calling SDK cleanup. Add `__enter__/__exit__`.

### [D3-F3] Bare Bracket Config Access Will Crash on Missing Keys
- **Files**: `binary_star_orchestrator.py:71,79`, `audit_controller.py:144,162`, `order_executor.py:627`, `loader.py:28-49`, `calculate_qty.py:73`
- **Description**: 8+ locations use `config["key"]` without `.get()` fallback. Triple-nested access `self.config['analysis_window']['micro_context']` is especially fragile.
- **Impact**: Malformed YAML → hard crash mid-session. If `trade_management` key is missing in `order_executor.py`, entire position protection fails.
- **Recommendation**: Replace with `.get()` with explicit fallbacks. Add startup config validation.

### [D3-F4] Stop-Loss Breach Check Uses Float Comparison Without Epsilon
- **File**: `src/agent/order_executor.py:328-330, 672`
- **Description**: `current_price <= sl` and `price_delta <= 0` use raw float comparisons. `price_delta == 0` guard in `calculate_qty.py:21` used for division safety.
- **Impact**: Floating-point rounding can cause missed stop-loss breaches or near-infinite quantities from division by ~0.
- **Recommendation**: Use `abs(current_price - sl) < epsilon` and `price_delta < 1e-12`.

### [D3-F5] Naive `datetime.now()` in Production Paths
- **Files**: `binary_star_orchestrator.py:390`, `email_notifier.py:105`, `run_evolution.py:66,170`, `run_sniper.py:138`
- **Description**: 5+ locations call `datetime.now()` without timezone, while 35+ other locations correctly use `datetime.now(timezone.utc)`.
- **Impact**: Mixed timezone-aware and naive timestamps in audit trails. Unreliable event sequence reconstruction in cross-region deployments.
- **Recommendation**: Replace all `datetime.now()` with `datetime.now(timezone.utc)`.

### [D3-F6] Signal Detector Failures Logged at DEBUG (Invisible in Production)
- **File**: `src/sniper/trigger.py:1306-1307`
- **Description**: The 14-signal detector loop catches `except Exception` and logs at `logger.debug`.
- **Impact**: With default INFO-level logging, ALL signal detector failures are invisible. Engine runs with silently degraded signal set.
- **Recommendation**: Log at `logger.warning`. Add per-detector failure counters with alerts.

### [D3-F7] Data Fetch Failures Return `[]` Masking Root Cause
- **Files**: `binance/client.py:146-148, 265-267, 303-305`
- **Description**: Multiple fetch methods catch `except Exception` broadly and return `[]`. Callers cannot distinguish "no data" from "fetch failed." `fetch_historical_klines` catches even `KeyboardInterrupt`.
- **Impact**: Downstream computation continues with empty data. Silent degradation of market analysis.
- **Recommendation**: Re-raise `KeyboardInterrupt`/`SystemExit`. Return `None` on failure so callers can distinguish.

### Sniper / Order Management

### [D4-F3] Cancel-Then-Replace — Position Quantity Not Re-Verified After Cancel
- **File**: `src/agent/order_executor.py:572-583`
- **Description**: `_optimize_same_direction` cancels orders at line 572, then places new OCO at line 583 using `net_qty` captured at line 529 (before cancel). Unlike `_migrate_trailing_stop` which re-fetches position after cancel.
- **Impact**: Partial fill during cancel window → OCO placed for wrong quantity → exchange rejection or oversold exit.
- **Recommendation**: Re-fetch position after cancel, mirroring line 494 pattern.

### [D4-F4] Position Re-Verification Returns None After Cancel — No Emergency Fallback
- **File**: `src/agent/order_executor.py:494-497`
- **Description**: After `cancel_all_symbol_orders` removes all protection, `get_symbol_position` returns `None` (transient API error). Code logs error and returns — leaving position naked.
- **Impact**: Transient API blip at worst possible moment → unprotected position.
- **Recommendation**: Call `execute_market_close` when `pos is None` after successful cancel.

### [D4-F5] Cancel-Replace Exception Handler Does Not Emergency Close
- **File**: `src/agent/order_executor.py:519-521`
- **Description**: Any non-`ClientError` exception in cancel-replace (TypeError, KeyError, unexpected network error) is caught but does not emergency-close. Falls through to `return trade_state`.
- **Impact**: If cancel succeeded before the exception, position is naked for up to 2 minutes.
- **Recommendation**: Call `execute_market_close` in the exception handler.

### Security

### [D5-F2] Entire `data/` Directory Served as Static Files
- **File**: `src/dashboard/server.py:32-33`
- **Description**: `app.mount("/data", StaticFiles(...))` exposes all session JSONs, audit reports, kline images, and heartbeat files via predictable URLs.
- **Impact**: Unauthenticated access to complete trading history, strategy parameters, and operational data. Enables strategy reverse-engineering and timing attacks.
- **Recommendation**: Serve only specific safe subdirectories. Add auth middleware to `/data` routes.

### [D5-F3] User-Controlled `data_root` Enables Arbitrary Directory Targeting
- **Files**: `dashboard/api/sniper_run.py:65`, `session_run.py:142`, `backtest.py:395`, `audits.py:25`, `sessions.py:74`
- **Description**: Every dashboard API endpoint accepts unsanitized `data_root` query parameter used directly for filesystem paths. Only `sessions.py:164-167` has path traversal protection.
- **Impact**: Attacker can direct app to read/write from unintended locations.
- **Recommendation**: Validate `data_root` against allowlist. Apply `.resolve()` prefix checks consistently.

### Test Coverage

### [D6-F1] 59% of src/ Modules Have ZERO Tests
- **Description**: 32 of 54 non-init modules lack any dedicated test. Sniper (trigger.py, scout.py): 0 tests. Dashboard (7 modules): 0 tests. Exchange clients: 0 tests. Market observer, liquidation estimator, topography engine, simulation sampler, audit controller: 0 tests.
- **Impact**: Critical trading logic (ConfluenceEngine, Guardian, exchange integration) runs in production with no automated verification.
- **Recommendation**: Prioritize tests for sniper/trigger.py and order_executor.py error paths.

### [D6-F2] Boundary Tests: NaN, Inf, and Extreme Values Untested Across ALL Math Functions
- **Description**: Zero tests for NaN, Inf, None, overflow, or exact boundary thresholds across all math utilities. Only `test_atr_metrics_invalid_atr` tests a zero-ATR scenario.
- **Impact**: Silent NaN propagation bugs (D1-F2, D1-F3, D1-F5, D1-F7, D4-F9) would have been caught with boundary tests.
- **Recommendation**: Add parametrized NaN/Inf/None/zero tests for every math function.
